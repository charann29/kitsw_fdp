# Faculty Task Management System - Submission Guidelines

## Submission Format Requirements

### Document Type
- **Primary Format**: Microsoft Word Document (.docx)
- **Alternative Format**: PDF (.pdf)
- **Content**: All screenshots with detailed explanations

### Document Structure

#### 1. Cover Page
- Project Title: "Faculty Task Management System"
- Student Name and ID
- Course/Subject Name
- Submission Date: July 12, 2025
- Institution Name

#### 2. Project Overview Section
- Brief project description (2-3 paragraphs)
- Tech stack used
- Key features implemented

#### 3. Backend Implementation Screenshots
Include screenshots of:
- **Project folder structure** (backend directory)
- **Database connection setup** (db.js file)
- **Task model schema** (Task.js file)
- **Controller functions** (taskController.js file)
- **API routes** (taskRoutes.js file)
- **Server setup** (server.js file)
- **MongoDB database** (showing collections and sample data)

#### 4. API Testing Screenshots
Include screenshots of:
- **POST /api/tasks** - Creating new tasks
- **GET /api/tasks** - Retrieving all tasks
- **GET /api/tasks/:id** - Retrieving single task
- **PUT /api/tasks/:id** - Updating a task
- **DELETE /api/tasks/:id** - Deleting a task
- **GET /api/tasks/status/:status** - Filtering by status
- Use Thunder Client or Postman for API testing

#### 5. Frontend Implementation Screenshots
Include screenshots of:
- **Project folder structure** (frontend directory)
- **Component files** (TaskCard.jsx, TaskForm.jsx)
- **Page components** (Dashboard.jsx, AllTasks.jsx, AddTask.jsx, EditTask.jsx)
- **Service file** (taskService.js)
- **App.js** and routing setup

#### 6. Application Interface Screenshots
Include screenshots of:
- **Dashboard page** - showing task summary
- **All Tasks page** - table view with edit/delete buttons
- **Add Task page** - form interface
- **Edit Task page** - pre-filled form
- **Filter view** - pending, completed, overdue tabs
- **Responsive design** - mobile/tablet views (if implemented)


### Screenshot Guidelines

#### Image Quality
- **Resolution**: Minimum 1920x1080 for desktop screenshots
- **Format**: PNG or JPEG (PNG preferred for code screenshots)
- **Clarity**: Ensure all text is readable
- **Zoom**: Adjust zoom level for better readability (125%-150%)
